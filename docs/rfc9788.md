# RFC 9788 – Header-Verschlüsselung

## Warum Header-Verschlüsselung?

E-Mails enthalten sensible Metadaten in den Headern:

- **Betreff** – Kann vertrauliche Informationen enthalten
- **Von / An / CC** – Kommunikationsbeziehungen und -netzwerke
- **In-Reply-To / References** – Thread-Zusammenhänge
- **Message-ID** – Eindeutig identifizierbare Nachrichtenreferenzen

Ohne Header-Schutz liegen diese Daten auch bei verschlüsseltem Nachrichtenkörper im Klartext vor und können von jedem SMTP-Server auf dem Übertragungsweg eingesehen werden.

## RFC 9788 in PrivMail

[RFC 9788](https://www.rfc-editor.org/rfc/rfc9788.html) definiert einen Standard zur Ende-zu-Ende-Verschlüsselung von E-Mail-Headern.

### So funktioniert es

1. **Senden**: Der Absender verschlüsselt die sensitiven Header-Felder (Subject, From, To, CC, Date, Message-ID, In-Reply-To, References) mit dem öffentlichen OpenPGP-Schlüssel des Empfängers.

2. **Äußere Struktur**: Die E-Mail erhält eine geschützte MIME-Struktur:
   - `multipart/encrypted` mit äußerem und innerem Teil
   - Der äußere Subject wird auf `[...]` gesetzt (keine Metadaten-Leakage)
   - Header `X-PrivMail-Header-Protection: RFC9788` kennzeichnet die geschützte E-Mail

3. **Empfang**: Der Empfänger entschlüsselt die geschützten Header mit seinem privaten Schlüssel und stellt die originalen Header-Felder im Webmail wieder her.

### Nachrichtenstruktur

```
MIME-Version: 1.0
Subject: [...]
X-PrivMail-Header-Protection: RFC9788
Content-Type: multipart/encrypted; protocol="application/pgp-encrypted"

--boundary
Content-Type: application/pgp-encrypted

Version: 1

--boundary
Content-Type: application/octet-stream

-----BEGIN PGP MESSAGE-----
(verschlüsselte Header + Body)
-----END PGP MESSAGE-----

--boundary--
```

### Sendeablauf

1. Empfänger und dessen öffentlichen Schlüssel ermitteln
2. Sensitive Header extrahieren und mit Empfänger-Schlüssel verschlüsseln
3. Äußeren Betreff auf `[...]` setzen
4. `X-PrivMail-Header-Protection: RFC9788` Header hinzufügen
5. Multipart/encrypted Struktur aufbauen
6. E-Mail über SMTP versenden

### Empfangsablauf

1. Eingehende E-Mail auf `X-PrivMail-Header-Protection: RFC9788` prüfen
2. Multipart/encrypted Struktur parsen
3. Geschützte Header mit eigenem privatem Schlüssel entschlüsseln
4. Originale Header-Felder wiederherstellen
5. E-Mail mit vollständigen Headern im Webmail anzeigen
